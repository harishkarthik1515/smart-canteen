Thank you for your interest in contributing to smart-canteen! We welcome improvements, bug fixes, documentation updates and new features. Please follow these guidelines to help us review and accept your contribution quickly.

1. Code of Conduct
- By contributing you agree to follow the project's Code of Conduct. See CODE_OF_CONDUCT.md for details.

2. How to contribute
- Fork the repository
- Create a new branch from main: git checkout -b feature/short-description
- Make your changes with clear, small commits
- Keep changes focused â€” one feature or fix per PR

3. Commit messages
- Use clear, imperative commit messages (e.g., "Add login validation")
- Reference issues when applicable (e.g., "Fixes #42")

4. Pull requests
- Open a pull request against the main branch
- Describe the problem you're solving and any relevant context
- Add screenshots or logs if relevant
- Link to any related issues

5. Tests
- Add or update tests for new features or bug fixes where applicable
- Run existing tests before submitting a PR

6. Coding style
- Follow existing project conventions. If the project uses a linter, run it and fix issues before submitting.

7. Questions
- If you're unsure where to start, open an issue with a proposal and we can help guide you.
